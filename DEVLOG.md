# Developer Log (DEVLOG)

This document tracks the execution flow, quick pivots, and technical roadblocks encountered while building SpendScope.

## Initial Setup & Landing Page
- Bootstrapped with Next.js 15 App Router and `shadcn/ui`.
- Opted for a "Voi-inspired" design system (coral red, deep dark mode, high contrast).
- **Roadblock**: Dark mode was causing text in the `.card-white` utility classes to become invisible because it inherited `--foreground` (which turns white in dark mode) against a forced white background.
- **Fix**: Leveraged Tailwind's powerful custom property system to strictly override `--foreground` and `--muted-foreground` inside `.card-white` specifically so the text always remains dark regardless of the global theme.

## The Audit Engine
- Initially considered using a dynamic LLM prompt to generate the entire audit. Realized very quickly that this would result in a fragile product. If the prompt hallucinates a feature or price, the entire product loses trust.
- **Pivot**: Built a strictly deterministic TypeScript engine (`lib/audit/engine.ts`). Hand-coded a robust pricing data structure (`lib/audit/pricing-data.ts`). This ensures every calculation is testable and completely defensible.

## The AI Integration
- The initial requirement was to use Anthropic's Claude API. 
- **Pivot**: Switched to Google's Gemini 1.5 Flash due to accessible API limits and fast inference speed during development. To ensure this didn't break the architecture, I maintained a provider-agnostic wrapper pattern.
- **Resilience**: Implemented a graceful degradation strategy. If the Gemini API times out (15s limit) or fails due to network issues, the app gracefully falls back to a deterministic summary generated by the rules engine, guaranteeing the user never sees a broken UI.

## Database & Lead Capture
- Set up Supabase PostgreSQL.
- **Decision**: Separated `audits` and `leads` into two tables with a foreign key constraint. This keeps the public report queries fast and clean (querying only `audits`) without accidentally exposing the `leads` table data.
- Encountered a TypeScript inference issue where `@supabase/supabase-js` inferred `never[]` for insert payloads due to extreme strictness in the generated types vs the JSON payload.
- **Fix**: Explicitly bypassed the generic type inference inside the database helpers (`lib/db/audits.ts`) by casting the payload during the `.insert()` operation, preventing brittle build failures while maintaining runtime safety.

## Transactional Emails
- Integrated Resend for transactional confirmation emails.
- **Roadblock**: The `from` address was initially set to a dummy domain (`spendscope.app`), causing Resend to silently drop the emails in development because the domain was unverified.
- **Fix**: Pivoted to using `onboarding@resend.dev` (Resend's default testing domain). This enables immediate testing of the email flow by sending to the registered developer's email address without waiting for DNS propagation.

## Final Polish
- Built the viral loop. Generated unique slugs on save.
- Added dynamic Open Graph metadata using Next.js `generateMetadata` so shared reports render beautifully on social platforms.
